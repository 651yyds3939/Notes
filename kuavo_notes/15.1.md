
# 人形机器人强化学习逻辑分析

## 整体逻辑：一条「仿真 MDP + PPO」流水线

这个项目是 Isaac Lab 的 `ManagerBasedRLEnv`（命令/观测/动作/奖励/终止/事件/课程 分模块管理） 加上 RSL-RL 的 PPO（`OnPolicyRunner`）。数据流可以概括为：

1. **命令 (Commands)**：例如 `base_velocity`，周期性采样期望线速度、角速度/航向。
2. **动作 (Actions)**：策略输出关节目标（这里是 `JointPositionActionCfg`，带 scale），经执行器作用到机器人。
3. **仿真步进**：Isaac 物理 + 传感器（接触力、射线高度等）。
4. **观测 (Observations)**：拼成 policy / critic 向量（Kuavo 在 rough 里对 critic 加了很多项）。
5. **奖励 (Rewards)**：每个 `RewTerm` 对应一个标量项，乘 `weight` 后由 Isaac Lab 的 `RewardManager` 求和为每步总奖励。
6. **终止 (Terminations)**：超时、非法接触、关节限位等结束 episode。
7. **事件 (Events)**：重置位姿/关节、域随机化、间歇推扰等。
8. **课程 (Curriculum)**：例如按跟踪奖励调节地形难度（`mdp/curriculums.py` 里会读 reward 的 episode 累计）。

强化学习算法侧：`train.py` 用 `RslRlVecEnvWrapper` 包一层后交给 `OnPolicyRunner`，内部是标准 **on-policy rollout → PPO 更新**（见 `rsl_rl/rsl_rl/runners/on_policy_runner.py`、`algorithms/ppo.py`）。奖励只出现在环境 `step` 的返回值里，算法本身不定义任务奖励。

## 「从 0 开始训练」具体指什么

入口是 `scripts/rsl_rl/train.py`：
* 先 `AppLauncher` 拉起 Isaac Sim；
* `import ext_template.tasks` 完成 Gym 环境注册（任务名 → `env_cfg_entry_point` + `rsl_rl_cfg_entry_point`）；
* Hydra 按 `--task` 载入对应 `EnvCfg` 和 PPO 的 `RunnerCfg`；
* `gym.make(args_cli.task, cfg=env_cfg)` 创建环境；
* `runner = OnPolicyRunner(...)`，默认随机初始化策略网络；
* `runner.learn(..., init_at_random_ep_len=True)` 开始采样与更新。

只有在 agent 配置里打开 `resume` 并指定 `load_run` / `load_checkpoint` 时，才会 `runner.load(resume_path)` 从 checkpoint 接着训；否则就是「从零」的随机策略 + 环境交互。

任务注册示例 (Flat / Rough Kuavo) 在：
`__init__.py` Lines 13-40
```python
gym.register(
    id="Legged-Isaac-Velocity-Flat-Kuavo-S42-v0",
    entry_point="omni.isaac.lab.envs:ManagerBasedRLEnv",
    disable_env_checker=True,
    kwargs={
        "env_cfg_entry_point": f"{__name__}.flat_env_cfg:KuavoS42FlatEnvCfg",
        "rsl_rl_cfg_entry_point": f"{agents.__name__}.rsl_rl_ppo_cfg:KuavoS42FlatPPORunnerCfg",
    },
)
// ... Rough 任务指向 rough_env_cfg:KuavoS42RoughEnvCfg ...

```

## 奖励相关：哪些文件「重要」

**1. 本仓库里真正写「奖励公式」的 Python**

* `exts/ext_template/ext_template/tasks/locomotion/velocity/mdp/rewards.py`
人形/双足相关的自定义项都在这里实现，例如：`feet_air_time_clip`、`feet_slide`、`track_lin_vel_xy_yaw_frame_exp`、`track_ang_vel_z_world_exp`、`contact_forces`、`stand_still_without_cmd`、`gravity_aligned_when_stopping`（README 里提到的停步姿态项）、`joint_power_l2`、`action_smoothness_l2` 等。

**2. 决定「用哪些项、权重多少、传什么参数」的配置（比 rewards.py 更常改）**

* `exts/ext_template/ext_template/tasks/locomotion/velocity/config/s42/rough_env_cfg.py` 的 `RewardsCfg`：Kuavo rough 地形训练的完整奖励表（跟踪、惩罚、足端、姿态、停步等），是实际训练 Rough 任务时最核心的配置面。
* `exts/ext_template/ext_template/tasks/locomotion/velocity/config/s42/flat_env_cfg.py`：`KuavoS42FlatEnvCfg` 继承 `KuavoS42RoughEnvCfg`，主要改地形/观测等，奖励只做了少量 override（例如线/角速度跟踪权重），其余项沿用 rough。

**3. 模板级参考（与 Kuavo 并行存在的通用 legged 示例）**

* `exts/ext_template/ext_template/tasks/locomotion/velocity/velocity_env_cfg.py` 中的 `RewardsCfg`：更偏 Isaac Lab 模板的 rough 速度跟踪组合；Kuavo 的 `rough_env_cfg` 继承的是 `omni.isaac.lab_tasks` 里的 `LocomotionVelocityRoughEnvCfg`，但奖励实现仍大量走本地 `mdp`，以 `rough_env_cfg` 为准。

**4. 大量「标准」奖励项来自 Isaac Lab 本体（不在本仓库里）**
`mdp/__init__.py` 第一行是：

`__init__.py` Lines 1-9

```python
"""This sub-module contains the functions that are specific to the locomotion environments."""

from omni.isaac.lab.envs.mdp import * # noqa: F401, F403

from .curriculums import * # noqa: F401, F403
from .events import * # noqa: F401, F403
from .observations import * # noqa: F401, F403
from .rewards import * # noqa: F401, F403
from .terminations import * # noqa: F401, F403

```

因此像 `lin_vel_z_l2`、`joint_torques_l2`、`illegal_contact` 等若未在本地 `rewards.py` 重新定义，实现都在 `omni.isaac.lab.envs.mdp`（安装好的 Isaac Lab 里）。调权重时你在 `RewardsCfg` 里改即可。

**5. 与奖励强相关、但不是「奖励函数」源码**

* `exts/ext_template/ext_template/tasks/locomotion/velocity/mdp/curriculums.py`：用地形难度课程时，会读 `track_lin_vel_xy_exp` / `track_ang_vel_z_exp` 的 episode 累计奖励来决定是否升/降地形等级（逻辑与奖励挂钩，函数本身在 `curriculums.py`）。

## 小结表

| 角色 | 路径 |
| --- | --- |
| 训练入口、是否 resume | `scripts/rsl_rl/train.py` |
| 任务与配置入口 | `exts/ext_template/ext_template/tasks/locomotion/velocity/config/s42/__init__.py` |
| Kuavo 奖励项与权重 (主) | `.../config/s42/rough_env_cfg.py` → `RewardsCfg` |
| Flat 上奖励微调 | `.../config/s42/flat_env_cfg.py` |
| 本仓库自定义奖励数学 | `.../mdp/rewards.py` |
| Isaac 自带 MDP 项 (含许多通用 reward) | 安装环境中的 `omni.isaac.lab.envs.mdp` |
| 课程用奖励统计 | `.../mdp/curriculums.py` |

若你接下来想改行为（例如停步、落脚重、转向），通常顺序是：先改 `rough_env_cfg.py` 的 `RewardsCfg`（权重/开关），需要新公式时再在 `mdp/rewards.py` 增加函数并在 `RewardsCfg` 里挂一条 `RewTerm`。
