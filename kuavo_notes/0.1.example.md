# 🤖 Kuavo 机器人二次开发实战与全离线部署指南

## 〇、 系统底座：双机分布式算力架构

为了彻底解决上位机显存爆炸的死穴，本指南所有项目均建立在**“双机分布式 ROS2 DDS 通信”**架构之上。严禁将所有计算载荷堆叠在单一节点。

### 硬件定责与资源分配
* **下位机（认知与决策中枢）：摩方 i9-13900H 工控机**
    * **配置**：14核20线程 CPU / 64GB DDR5 内存 / 500G SSD / Iris Xe 核显。
    * **载荷**：负责承载大语言/视觉模型（Qwen2-7B/LLaVA）、纯 Python 离线 TTS 引擎、MoveIt 3D 轨迹规划与解算、BehaviorTree.CPP 全局行为树、高频视觉检测（YOLOv8 OpenVINO版）以及建图定位算法。
* **上位机（感知与执行网关）：NVIDIA Jetson Orin NX**
    * **配置**：6 核 ARM CPU / 1024 CUDA / 8GB 统一内存。
    * **载荷**：彻底卸载大模型，仅运行 Faster-Whisper ASR 语音识别（榨干 GPU float16 算力）、传感器硬件驱动（Mid-360 激光雷达、Gemini-335L 深度相机）、执行下位机传来的关节运动指令，以及维持底层的官方经典控制（WBC/MPC）防摔平衡。

## 一、 二次开发全景项目矩阵 (共 15 项)

### 模块 A：视觉-语言-动作 (VLA) 与大模型交互

**1. 桌面具身 VLA 语义抓取**
* **项目描述**：打破传统的“坐标级”硬编码编程，让机器人具备语义级别的物体操作能力。用户只需下达模糊的自然语言指令（例如：“我有点渴了，把桌上能喝的递给我”），机器人需要自主完成“意图理解 $\rightarrow$ 目标视觉定位 $\rightarrow$ 空间避障规划 $\rightarrow$ 物理抓取”的完整闭环。
* **实现方法**：
    * **控制流切分**：采用 "Stop-and-Go" 策略。移动至桌前后，通过 ROS 节点发送指令锁死双腿，底盘彻底交由官方经典控制（WBC）接管以维持动态平衡。
    * **感知与推理**：Orin NX 获取 Gemini-335L 的 RGB 图像发给下位机，下位机的 Qwen-VL（视觉语言模型）推理出目标物体的类别与 2D 像素坐标。
    * **执行闭环**：结合深度相机的点云数据，将 2D 坐标转换为基座坐标系下的 3D 空间坐标 $(x, y, z)$。目标点传入驻留在 i9 上的 MoveIt 引擎，解算逆运动学（IK）并生成避障轨迹，最后发布到 `/joint_trajectory_controller` 驱动真机手臂抓取。

**2. 大模型 Prompt 驱动的“空间动作原语提取”**
* **项目描述**：彻底抛弃繁琐的中间层解析代码，将大语言模型（LLM）直接升格为机器人的“高级任务规划器（High-level Planner）”。利用 LLM 强大的逻辑拆解能力，让其在与用户对话的同时，隐式地输出可被机器人直接执行的底层动作代码标签。
* **实现方法**：
    * **Prompt 工程**：在下位机 Qwen2-7B 的 System Prompt 中注入严格的格式约束与“动作字典”。例如：“你现在是具备物理实体的机器人，必须在回复文本末尾输出动作标签。字典包含：`GRAB(target)`、`POINT(target)`、`NOD()`。”
    * **正则捕获与映射**：当下位机的 Python 主控流接收到大模型的流式或完整回复时，使用正则表达式提取 `[EXECUTE: ...]` 标签。
    * **动作触发**：提取出的标签直接映射到预先写好的 ROS2 Action Client 中，例如解析到 `GRAB(apple)`，即刻触发 MoveIt 抓取子程序，实现语言到动作的零延迟桥接。

**3. 桌面异常诊断与状态修复**
* **项目描述**：赋予机器人超越基础抓取的“物理世界常识（Commonsense Reasoning）”。机器人不再是指令驱动的盲目执行者，而是能够自主观察桌面，发现诸如“水杯倾倒”、“物品危险边缘悬空”、“垃圾散落”等异常物理状态，并主动实施纠正动作。
* **实现方法**：
    * **场景诊断**：调用视觉大模型对当前桌面图像进行推理，Prompt 设定为：“找出当前场景中不符合物理常理或处于危险状态的物体，并以 JSON 格式输出其名称与建议修复动作”。
    * **姿态矫正规划**：如果模型输出 `{"target": "cup", "status": "tipped", "action": "upright"}`，系统需结合点云数据计算倒伏水杯的位姿（Pose）。MoveIt 不仅要规划抓取目标，还需要附加一个末端执行器的旋转约束（如 Roll 轴旋转 90 度），引导机械臂完成“扶正”这一复杂操作。

**4. 视觉意图预测与主动递工具 (Human-Robot Collaboration)**
* **项目描述**：这是一个极具工业价值的人机协作（HRC）项目。机器人在人类旁边充当装配助理，通过连续观察人类的细微肢体动作与当前任务状态，提前“预判”人类下一步需要的工具，并在人类开口前主动将工具递送至安全交接区。
* **实现方法**：
    * **动作时序检测**：在下位机运行 YOLOv8-Pose 或 MediaPipe 追踪人类手部关键点，结合目标检测识别当前人类正在操作的零件（如：正在寻找螺丝、手持未拧紧的螺栓）。
    * **意图预测**：将最近 5 秒的动作序列特征喂给本地大模型，推理出：“主人正在进行拧螺丝作业，需要螺丝刀”。
    * **安全递送规划**：定位桌上的螺丝刀后，MoveIt 规划一条避开人类手臂活动范围（通过动态设立碰撞圆柱体）的轨迹，将工具悬停在人类手边 15cm 的安全区域等待交接。

**5. 行为自检与情感语音日志**
* **项目描述**：将冰冷枯燥的系统 Debug 日志转化为拟人化的高维交互体验。机器人在长时间的联调或工作结束后，能够自主梳理系统数据，并以符合其设定的“人设（如傲娇、疲惫的打工人）”进行语音播报总结。
* **实现方法**：
    * **数据聚合**：开发一个 ROS2 后台守护节点，订阅 `/rosout`、`/diagnostics` 和底盘 IMU 话题。统计一整天的“不倒翁防摔触发次数”、“伺服电机过热警告次数”以及“视觉抓取成功率”。
    * **文本生成与合成**：将清洗后的结构化数据转为自然语言提示词传入 Qwen2-7B，例如：“你今天摔倒拦截了 5 次，电机发热 2 次，请用疲惫打工人的语气向主人抱怨并汇报”。最后由下位机 5000 端口的 VITS 引擎进行极具情感的语音播报。

### 模块 B：高级控制、操作与安全性

**6. 基于“虚拟弹簧”的柔性导纳交互**
* **项目描述**：针对官方底层 C++ 控制闭源、无法直接修改电机力控算法的痛点，在应用层（Python/ROS2）开发一套伪力控系统（Cartesian Admittance Control）。实现机器人手臂能随人手“隔空牵引”移动，或在感受到阻力时顺从退缩的柔性安全交互。
* **实现方法**：
    * **弹簧力学建模**：开启深度相机，利用 MediaPipe 提取人类手掌的三维空间坐标 $P_{human}$。同时读取机器人末端执行器的坐标 $P_{robot}$。
    * **虚拟力计算**：在控制节点中构建虚拟胡克定律模型 $F = k \cdot (P_{human} - P_{robot})$，其中 $k$ 为虚拟刚度系数。
    * **运动学映射**：将计算出的虚拟力 $F$ 结合阻尼项，转换为机器人末端执行器的目标位置增量 $\Delta P$，通过逆运动学接口高频下发，从而在没有任何实体力传感器的情况下，利用视觉闭环实现“顺滑跟随”的柔性错觉。

**7. 桌面多目标“推移滑动”操作 (Non-prehensile)**
* **项目描述**：针对二指夹爪（最大行程限制及 180N 夹持力）无法抓取过大、过重或易碎物体的物理局限，开发非抓取式（Non-prehensile）操作技能，让机械臂通过“推（Pushing）、拨（Poking）”的方式完成桌面物品的整理与归放。
* **实现方法**：
    * **平面提取与投影**：使用 PCL（点云库）结合 RANSAC 算法提取桌面的法向量及平面方程，将目标物体的点云中心投影至桌面上。
    * **推移轨迹生成**：不使用 MoveIt 的抓取 API，而是生成一条严格平行于桌面（保持 $Z$ 轴高度恒定）的笛卡尔直线轨迹（Cartesian Path）。控制机械臂以闭合状态的夹爪背面作为接触面，将目标物体匀速推入指定的收纳区域。

**8. 被动推搡对抗与动态足端修正**
* **项目描述**：强化现有的“原地 RL 不倒翁”功能。当机器人受到超出静止摩擦力和关节扭矩极限的猛烈推搡时，单纯依靠上半身扭动已无法维持重心，系统必须能在瞬间切出 RL，触发经典控制的“迈步防摔”机制。
* **实现方法**：
    * **状态监控与中断**：高频（1000Hz）读取 Orin NX 上的 IMU 角速度与横滚/俯仰角。设定极值阈值，一旦突破，立即挂起当前运行的 RL 策略网络。
    * **捕获点（Capture Point）计算**：依据经典倒立摆模型，计算瞬时捕获点公式 $x_{cp} = x + \dot{x}\sqrt{h/g}$，得出为保持平衡所需的落脚点偏置。
    * **API 劫持切换**：调用官方闭源的单步移动 API 向目标点快速迈出碎步，待 IMU 数据回落至安全区间后，立即将控制权交还给强化学习策略，完成全被动抗扰测试。

**9. 基于点云的桌面 3D 机械臂避障规划**
* **项目描述**：将宏观的底盘建图导航能力浓缩至微观桌面。当桌面上存在水杯、书籍等复杂障碍物时，机械臂在执行抓取时能够自动在 3D 空间中计算出一条安全绕行的无碰撞弧线。
* **实现方法**：
    * **环境三维重建**：在 Orin NX 上获取 Gemini-335L 的高频点云流，经过体素滤波（Voxel Grid Filter）下采样后，输入到 Octomap 节点生成实时更新的 3D 八叉树碰撞地图。
    * **MoveIt 场景接入**：将 Octomap 发布为 MoveIt 的 `PlanningScene`。在执行动作时，MoveIt 底层的 OMPL 规划库（如 RRTConnect 或 PRM 算法）会自动将这些体素块视为绝对障碍区，解算出在各个关节限位内完美避障的 C-Space（构型空间）轨迹。

**10. 模仿学习 + 强化学习全身协同复现**
* **项目描述**：挑战高动态控制极限，让机器人重现人类的高难度动作（如打太极、武术姿态），同时解决直接播放轨迹导致机器人重心失衡摔倒的问题。实行“上肢模仿追踪，下肢死守平衡”的全身协同控制（Whole-Body Control）。
* **实现方法**：
    * **动作重定向**：从 AMASS 或 Motion-X 数据集中提取人类 SMPL 动捕数据，通过骨骼映射算法（Motion Retargeting）转换为 Kuavo 的 URDF 关节角度序列。
    * **仿真强化炼丹**：在 Isaac Gym/Lab 中搭建训练环境。设计复合奖励函数：给予上半身关节极高的“轨迹跟踪奖励（Tracking Reward）”，同时对下半身施加严苛的“质心（CoM）偏离惩罚”与“足端滑动惩罚”。训练出的策略能够在努力模仿手部动作的同时，双腿疯狂微调抗拒重心倾翻，最后通过 Sim-to-Real 部署到真机。

### 模块 C：遥操作与数字基建

**11. 低成本单目体感遥操作**
* **项目描述**：摆脱昂贵的动捕服和 VR 设备限制，仅利用一台插在下位机上的普通电脑摄像头或串流手机，实现人类操作员与机器人上半身姿态的毫秒级同步映射（Teleoperation）。
* **实现方法**：
    * **骨骼点提取**：在下位机运行轻量级的 MediaPipe Pose 框架，实时输出操作员在二维/三维空间下的 33 个身体关键点。
    * **相对角度推算**：在 Python 中利用向量内积和叉乘算法，计算出肩膀的俯仰/横滚角（Pitch/Roll）、手肘的偏航角等核心相对角度。
    * **实时映射约束**：对角度进行低通滤波平滑处理，映射到机器人的关节转角限位内，通过 ROS `/joint_commands` 直接下发。并在代码层加入最大速度限制墙，防止人类动作过猛导致机器人自伤。

**12. 3D 仿生眼追踪与多模态情感吐槽**
* **项目描述**：彻底激活机器人的“头部交互”生命力。在四肢锁死的情况下，机器人的脖颈关节会如影随形地死死盯住实验室中走动的人，并在对视时结合本地视觉大模型进行专属的幽默语音吐槽。
* **实现方法**：
    * **视觉伺服（Visual Servoing）**：通过 MediaPipe FaceMesh 持续获取人脸在画面中的像素中心点。利用 PID 控制器计算像素偏差，将其转化为脖颈两个舵机（Pan & Tilt）的转角指令，实现平滑追踪。
    * **多模态链路**：当判断人脸居中停留超过 2 秒，触发“快门”截图，送入 Qwen-VL 并附带吐槽 Prompt（如“观察此人的穿着并用傲娇的语气锐评”）。结果回传至 TTS 引擎进行物理发声。

**13. C++ 具身全局行为树调度 (BehaviorTree.CPP)**
* **项目描述**：进行底层架构的现代化升级，彻底废弃用 Python 编写的冗长、易死锁、难调试的 `if-else` 状态机。引入 ROS2 工业界标配的 `BehaviorTree.CPP` 框架，打造机器人的最高统帅部。
* **实现方法**：
    * **节点封装**：将所有能力模块用 C++ 封装为行为树节点。如将 LLM 推理封装为异步执行的 `CoroActionNode`，将电池检测封装为 `SyncActionNode`，将 MoveIt 抓取封装为带超时打断的节点。
    * **逻辑编排**：使用 XML 脚本编排优先级队列（Fallback）和顺序执行流（Sequence）。并利用 Groot 可视化工具实时监控运行时节点状态，确保系统在复杂任务并行时（如边听指令边避障）的强鲁棒性。

**14. 手势/语音多模态安全密码锁**
* **项目描述**：在进行高风险的 MoveIt 机械臂实机调试前，构建一道基于物理行为和多模态验证的“系统安全隔离墙”。防止杂音或错误指令意外触发机械臂暴走砸坏桌面。
* **实现方法**：
    * **休眠与警戒**：在全局行为树的根节点下放置一个前置的控制流阻塞节点，此时机器人所有运动神经（控制话题发布）被切断。
    * **解密验证**：循环运行 YOLOv8 手势识别节点与轻量级语音唤醒节点。只有当视觉模型检测到特定的“安全握拳”手势，并且 ASR 监听到特定的“唤醒词”时，状态机才跳转至 `ACTIVE` 工作模式，释放 API 锁。任何异常中断均会使其瞬间回退至 `LOCKED` 状态。

**15. 局域网全动态 3D 数字孪生可视化面板**
* **项目描述**：由于 Orin NX 显存极度紧张，被迫通过 `multi-user.target` 剥离了图形桌面，导致调试时无法使用 Rviz 观察机器人内部状态。本项目旨在下位机或外部开发机上建立一套 Web 端的数字孪生工作站。
* **实现方法**：
    * **后端桥接**：运行 `rosbridge_suite`，将 ROS2 的 `/tf` 坐标系转换、`/joint_states` 关节数据以及 `/planned_trajectory` 路径规划等核心话题通过 WebSocket 广播。
    * **前端渲染**：开发 Web 端页面，引入 `three.js` 与 `ros3djs` 库，在浏览器中解析 Kuavo 的 URDF 文件。实现真机上的每一次扭动、MoveIt 生成的每一条绿色预规划轨迹，都能以 3D 模型的方式在网页上以 60fps 高保真同步渲染，彻底解决“瞎子调试”的痛点。

## 二、 运行注意事项与极限避坑指南

### 1. 内存死穴与“无头模式”续命法则
* **问题**：Orin NX 8G 内存无法承受视觉处理、ASR 和图形桌面的并发压力。
* **强制执行**：在 Orin 终端必须执行 `sudo systemctl isolate multi-user.target` 剥离 Ubuntu GUI 桌面引擎。释放出的近 1GB 纯血内存是保障 ASR 语音不卡死、系统不报 OOM 的生命线。

### 2. MoveIt 抓取必须依托经典控制的 WBC 护航
* **动作铁律**：严禁在 RL 模式走路时使用 MoveIt。
* **原理解析**：机器人必须采取 "Stop-and-Go" 停走策略。手臂前伸会改变重心导致倾翻。当底盘切换为官方经典控制模式时，其内部极高频的 WBC（全身控制）和 MPC（模型预测控制）会实时感知前倾力矩，通过微调膝关节和踝关节死死保住动态平衡。

### 3. 高载荷传感器降频保命机制
* **深度相机 (Gemini-335L)**：抓取桌面静态物体时，**绝对禁止**使用 1280x800 @ 30fps。在 ROS 驱动层强制降频至 640x400 @ 10Hz，只在需要 LLM 推理时通过服务（Service）请求单帧高清单图。
* **激光雷达 (Mid-360)**：桌面抓取或原地互动项目时，暂停雷达的建图节点，只保留其内部的 1000Hz IMU 数据供底层平衡器使用，释放网络带宽。

### 4. 电爪热保护与电流自锁机制
* **操作禁忌**：配置的二指爪具有 180N 的巨大夹持力。在闭合抓取硬物（如杯子）后，**立刻切断电爪的维持电流**。利用其机械自锁特性保持抓持，严禁长时间高负载通电，否则极易导致末端执行器烧毁。

### 5. 局域网 DDS 穿透与跨主板通信
* **网络拓扑**：下位机（192.168.26.1）与上位机（192.168.26.12）之间必须通过千兆网线直连或专属 5G 路由器通信。
* **ROS2 环境变量**：确保两台机器的 `.bashrc` 中配置了完全一致的 `ROS_DOMAIN_ID`，以便 MoveIt 规划节点和硬件执行节点可以无缝穿透物理主板进行 Topic 交互。
